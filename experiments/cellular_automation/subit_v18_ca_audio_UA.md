## 3. Ядро симуляції

### 3.1 Модель клітинного автомату

| Параметр | Значення |
|----------|----------|
| **Топологія** | Тороїдальна сітка (циклічні межі) |
| **Розмір сітки** | 40–120 клітин (налаштовується, за замовчуванням 80×80) |
| **Сусідство** | Муре (8 сусідів: N, NE, E, SE, S, SW, W, NW) |
| **Стани клітин** | Бінарні: 0 (мертва), 1 (жива) |
| **Правила** | 8-бітні (0–255), кожна клітина має власне правило |

### 3.2 Правило переходу (B3/S23-подібне)

```
Для кожної клітини (x, y):
  1. Підрахувати живих сусідів (liveNeighbors ∈ [0, 8])
  2. Новий стан = (rule[cell] >> liveNeighbors) & 1
     • Біт 0 → мертва
     • Біт 1 → жива
  3. Визначити домінантне правило серед живих сусідів (голосування)
  4. Оновити правило клітини залежно від стану та мутацій
```

### 3.3 Еволюція правил

| Механізм | Опис | Ймовірність |
|----------|------|-------------|
| **Адаптація живих** | Наслідування домінантного правила сусідів з мутацією | Завжди |
| **Мутація живих** | Випадкове відхилення від домінантного правила | `±liveMutRange` |
| **Адаптація мертвих** | Мертві клітини копіюють правило сусідів | `deadAdaptRate` (0–100%) |
| **Глобальна мутація** | Повністю випадкове нове правило | `mutationRate` (0–5%) |

## 4. Метрики та аналітика

### 4.1 Основні метрики

| Метрика | Формула | Діапазон | Опис |
|---------|---------|----------|------|
| **Популяція** | `Σ states[i]` | [0, SIZE²] | Кількість живих клітин |
| **Ентропія правил (H_rule)** | `-Σ p(r) × log₂(p(r))` | [0, 8] | Різноманітність правил у популяції |
| **Середня зміна правила (Δrule)** | `Σ \|newRule - oldRule\| / pop` | [0, 255] | Швидкість еволюції правил |
| **Унікальні правила** | `\|{r : freq(r) > 0}\|` | [0, 256] | Кількість активних правил |
| **Густина** | `pop / SIZE² × 100%` | [0%, 100%] | Відсоток живих клітин |
| **Фітнес правила** | `children / parents` | [0, ∞) | Репродуктивний успіх |

### 4.2 Фазова класифікація (Ω)

| Фаза | Умова | Колір | Опис |
|------|-------|-------|------|
| **STABLE** | `Δrule < 0.5` AND `H_rule > 0.8` | 🟢 Зелений | Стабільна екосистема, мало змін, висока різноманітність |
| **CYCLIC** | `Δrule < 2.0` AND `H_rule > 0.4` | 🟡 Жовтий | Циклічна динаміка, помірна зміна |
| **CHAOTIC** | `Δrule > 5.0` | 🔴 Червоний | Хаотична еволюція, швидкі зміни правил |
| **META** | Все інше | 🔵 Фіолетовий | Перехідна/метастабільна фаза |

## 5. Візуалізація

### 5.1 Canvas-елементи

| ID | Розмір | Призначення | Оновлення |
|----|--------|-------------|-----------|
| `mainCanvas` | 400×400 | Фізичний простір: кольорові живі клітини | Кожні 3 покоління |
| `ruleCanvas` | 200×200 | Простір правил: карта правил (R, 255-R, 3R) | Кожні 3 покоління |
| `deltaCanvas` | 200×200 | Карта змін: інтенсивність = Δrule × 8 | Кожні 3 покоління |
| `transCanvas` | 256×256 | Граф переходів: правило → нове правило (теплова карта) | Кожні 3 покоління |
| `omegaPlot` | 500×150 | Траєкторія H_rule (зелена) та Δrule (червона) | Кожні 3 покоління |
| `phaseCanvas` | 200×200 | Фазовий простір: Δrule (X) vs H_rule (Y) | Кожні 3 покоління |
| `audioCanvas` | 500×80 | Аудіовізуалізатор (спектр частот) | Кожен кадр |

### 5.2 Кольорова схема

- **Живі клітини**: HSL відтінок залежно від правила (`hue = rule × 360 / 256`)
- **Фон**: `#0a0a0a` (темний)
- **Панелі**: `#141414` з радіусом 8px
- **Акцент**: `#2a5` (зелений) / `#f0a` (рожевий для аудіо)

## 6. Аудіосистема

### 6.1 Архітектура звуку

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Осцилятори │────→│  Gain/Pan   │────→│ Master Gain │
│  (FM/AM/    │     │  (стерео)   │     │ (гучність)  │
│  Additive)  │     │             │     │             │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                    ┌─────────────┐     ┌──────┴──────┐
                    │  Reverb     │←────│  Analyser   │
                    │ (Convolver) │     │ (FFT 256)   │
                    └─────────────┘     └─────────────┘
                                               │
                                          ┌────┴────┐
                                          │  Вихід  │
                                          │(колонки)│
                                          └─────────┘
```

### 6.2 Параметри синтезу

| Параметр | Діапазон | За замовчуванням | Опис |
|----------|----------|------------------|------|
| **Гучність** | 0–100% | 30% | Загальна гучність |
| **Базова частота** | 55–880 Hz | 110 Hz | Основний тон (A2) |
| **Масштаб частоти** | 0.1–3.0× | 1.0× | Множник базової частоти |
| **Розмір хорусу** | 1–8 | 3 | Кількість голосів |
| **Тип синтезу** | FM / AM / Additive / Drone | FM | Алгоритм генерації |
| **Реверберація** | On/Off | On | Просторовий ефект (2s імпульс) |

### 6.3 Типи синтезу

| Тип | Алгоритм | Тембр |
|-----|----------|-------|
| **FM** | Синус + модулятор частоти (0.5× основної) | Багатий, дзвінкий |
| **AM** | Sawtooth | Різкий, агресивний |
| **Additive** | Sine/Triangle alternate | Органоподібний |
| **Drone** | Чистий Sine | Постійний, медитативний |

### 6.4 Мапінг даних → музика

| Мапінг | Джерело даних | Музичний параметр |
|--------|---------------|-------------------|
| **Ентропія** | H_rule → кількість голосів | Кількість осциляторів |
| | Δrule × 10 → detune | Відхилення висоти |
| **Популяція** | Популяція/SIZE² → гучність | Амплітуда master gain |
| | Правила → пентатоніка/мінор | Акордова структура |
| **Фаза** | Фаза Ω → лад (pentatonic/minor/chaotic) | Гамма |
| | Δrule × 20 → зсув ноти | Модуляція висоти |

### 6.5 Музичні лади

| Фаза | Лад | Інтервали (semitones) |
|------|-----|----------------------|
| STABLE | Пентатоніка мажор | 0, 2, 4, 7, 9, 12, 14, 16, 19, 21, 24 |
| CYCLIC | Мінор | 0, 2, 3, 5, 7, 8, 10, 12, 14, 15, 17, 19, 20, 22, 24 |
| CHAOTIC | Хаотичний | 0, 1, 3, 4, 6, 7, 9, 10, 12 |
| META | Пентатоніка | Як STABLE |

## 8. Технічні деталі

### 8.1 Структури даних

```javascript
// Основні масиви (TypedArrays для продуктивності)
states:      Uint8Array  // SIZE × SIZE, 0/1
rules:       Uint8Array  // SIZE × SIZE, 0–255
prevRules:   Uint8Array  // Попередні правила
deltaRules:  Uint8Array  // |newRule - oldRule|
nextStates:  Uint8Array  // Буфер для наступного стану
nextRules:   Uint8Array  // Буфер для наступного правила
totalTransition: Uint32Array  // 256 × 256, граф переходів

// Метрики
ruleFreq:    Uint32Array  // 256, частота правил
parentCount: Uint32Array  // 256, кількість батьків
childCount:  Uint32Array  // 256, кількість нащадків
fitnessArr:  Float64Array // 256, фітнес
HUE_TABLE:   Float64Array // 256, попередньо обчислені відтінки

// Історія
omegaHistory: Array<{hRule, avgDeltaRule, omega, omegaClass}>
```

### 8.2 Продуктивність

| Аспект | Реалізація |
|--------|------------|
| **Рендеринг** | `createImageData` + пряме маніпулювання пікселями |
| **Симуляція** | Оптимізовані вкладені цикли без функційних викликів |
| **Пам'ять** | Перевикористання масивів (двобуферна схема) |
| **Оновлення** | Роздільне: симуляція — кожен кадр, візуалізація — кожні 3 покоління |
| **FPS лічильник** | Оновлення раз на секунду |

### 8.3 Обмеження

| Параметр | Значення | Пояснення |
|----------|----------|-----------|
| Макс. історія Ω | 200 точок | Фіксований буфер для графіка |
| Макс. правил | 256 | 8-бітна адресація |
| Макс. сусідів | 8 | Муре-околиця |
| Аудіо контекст | Потребує взаємодії | Політика браузера (autoplay) |
